# Functional Specification (UI-centric)

## 1. Название модуля: Витрина (Home Page)
**Actor (Роли):** Гость, Авторизованный пользователь.
**Business Rules:**
*   *Трендовые товары:* Выборка топ-товаров (сейчас статично отбираются по логике `rating >= 4.7`).
*   *Выгодные предложения:* Выборка конкретных предложений (Offers), привязанных к товарам с лейблом "Выгодно".
*   *Поиск на карте:* Динамическая фильтрация списка магазинов по совпадению поискового запроса.
**State Mapping:**
*   **Loading:** Скелетоны для блоков Hero, Тренды, Акции. Блокирование инпута гео-поиска.
*   **Success:** Отображение карусели, карточек товаров, карты с маркерами продавцов.
*   **Empty:** Если нет акций — скрытие целого блока "Выгодные предложения". Если нет результатов на карте — вывод сообщения "Магазины не найдены".
*   **Error:** Тост-уведомление "Не удалось загрузить данные витрины", кнопка "Повторить попытку".
**Flow (Сценарий пользователя):** 
Загрузка главной -> Ознакомление с промо -> Клик на "Все предложения" -> Переход в Catalog (с предзаполненными параметрами `/?view=offers&sort=price_asc&label=deals`).

## 2. Название модуля: Каталог и Фильтры (Catalog Page)
**Actor (Роли):** Гость, Авторизованный пользователь.
**Business Rules:**
*   *Переключение режимов (Views):* Модели (Products), Предложения (Offers), Магазины (Sellers), Бренды (Brands).
*   *Синхронизация состояния:* Выбранные фильтры сохраняются в `window.State`, формируя общий контекст поиска без засорения строки URL после отработки Router-хита.
*   *Логика фильтрации:* Пересечение характеристик ("AND" между Бренд и Категория) и объединение ("OR" внутри массива Брендов).
*   *Сортировка:* По популярности, цене (asc/desc), рейтингу, новизне.
**State Mapping:**
*   **Loading:** Вывод скелетонов-карточек во время калькуляции фильтров. Блокировка контролов `select` и `input checkbox`.
*   **Success:** Обновленный грид 2-4 колонки карточек. Обновление хедера со счетчиком "Найдено: X".
*   **Empty:** Иллюстрация "🔍 Сущности не найдены", кнопка "Сбросить все фильтры".
*   **Error:** Блок "Ошибка применения фильтров", кнопка "Повторить".
**Flow (Сценарий пользователя):** 
Открытие каталога -> Открытие боковой панели фильтров (offcanvas на мобильных) -> Выбор параметров -> Клик "Применить" -> Обновление State -> Ререндер сетки.

## 3. Название модуля: Карточка Товара (Product Page)
**Actor (Роли):** Гость, Авторизованный пользователь.
**Business Rules:**
*   *Иерархия связей:* Модель (Product) -> Конфигурации/SKU (Variants) -> Физические Предложения продавцов (Offers).
*   Выбор модификации (память, цвет) обновляет список доступных в магазинах предложений и меняет "Стартовую цену" в хедере.
**State Mapping:**
*   **Loading:** Скелетон галереи, скелетон таба "Характеристики" и загрузочный лоадер списка предложений.
*   **Success:** Отрисовка контента продукта на основе выбранного SKU-варианта.
*   **Empty:** При отсутствии предложений по варианту — кнопка "Узнать о поступлении".
*   **Error:** Ошибка 404 "Товар не найден" -> редирект на главную/каталог.
**Flow (Сценарий пользователя):** 
Клик по карточке из каталога -> Загрузка данных модели -> Выбор цвета/памяти -> Изменение HashURL `?variant=X` -> Фильтрация предложений магазинов снизу от карточки.

## 4. Название модуля: Карточка Продавца (Seller Page)
**Actor (Роли):** Гость.
**Business Rules:**
*   Группировка каталога продавца секционно: Телефоны, Планшеты (по Классам и Категориям).
*   Дополнительное разбиение предложений вендора на вкладки "Модели" и "Предложения".
**State Mapping:**
*   **Loading:** Пульсирующие заглушки лоадеров обложки и аватарки.
*   **Success:** Вывод информации вендора (рейтинг, график работы, адрес) и его витрины товаров.
*   **Empty:** "Этот продавец еще не добавил товаров в каталог".
*   **Error:** "Магазин временно недоступен" (404).
**Flow (Сценарий пользователя):** 
Клик по лого продавца в списке офферов -> Загрузка профиля -> Переключение табов "Модели/Предложения".
