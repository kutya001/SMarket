# Project Manifest: SotkaMarket

## 1. Описание проекта
**SotkaMarket** — маркетплейс-агрегатор предложений от проверенных продавцов (магазинов) мобильной электроники, планшетов, гаджетов и аксессуаров. Бизнес-цель: предоставление пользователю единой витрины для подбора товара, сравнения цен от разных магазинов и поиска оптимального предложения.

## 2. Технологический стек
*   **Разметка:** HTML5 (семантическая).
*   **Стилизация:** Tailwind CSS (через CDN), Custom CSS (анимации, скроллбары, переопределение стилей карт).
*   **Логика:** Vanilla JavaScript (ES6+), модульная структура.
*   **Маршрутизация:** Клиентская маршрутизация (Hash-based, `window.location.hash`).
*   **Стейт-менеджмент:** Глобальный объект хранилища `window.State`, синхронизация с `localStorage`.
*   **Внешние интеграции:** Leaflet.js (интерактивные карты).
*   **PWA:** Service Worker (кеширование манифеста и статического контента для установки на устройства).

## 3. Архитектурные соглашения
Проект организован по принципам приближенным к Feature-Sliced Design / Component-Based:
*   `/src/js/pages/` — логика слоев инициализации экранов (Home, Catalog, Product, Offer, Seller).
*   `/src/js/components/ui/` — переиспользуемые UI-компоненты (ProductCard, OfferCard).
*   `/src/js/router/` — обработчик клиентских маршрутов (`hashchange`).
*   `/src/js/store/` — управление глобальным состоянием (`window.State`).
*   `/src/js/utils/` — инфраструктурные хелперы (карта, аналитика, утилиты).
*   `/src/js/data/` — моки сущностей.

**Правила написания кода для разработчиков:**
*   Отсутствие прямого управления DOM вне компонентов/страниц (модули генерируют HTML-шаблоны).
*   Строгое разделение состояния (State) и отображения (View).
*   Обработка событий через глобальные методы (`window.methods`) или явное делегирование.

## 4. Статус прототипа
Текущая версия — **UI-прототип с Mock-данными**.
*   **Подлежит замене на API:** Файл `/src/js/data/mockData.js` (PRODUCTS, OFFERS, SELLERS, BRANDS, CATEGORIES).
*   **Бэкенд:** Требуется разработка RESTful API (поиск, фильтрация, деталка карточки товара).
*   **Не реализованы (требуется бэкенд):** Пагинация (на уровне БД), SSR (сейчас выполняется полный клиентский рендер), серверная авторизация пользователей, корзина/оформление заказа (С текущим скоупом продукт — CPA-витрина).
